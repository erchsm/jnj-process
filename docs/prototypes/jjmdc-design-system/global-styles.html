In the prototype, adding the class "heading" to any heading tag causes that heading to assume a 900 font-weight (black), and sets the display to inline-block.

The "heading" class does NOT dictate the color of the heading, thus allowing it to be inherited from its parent element.

The "heading" class does NOT dictate the text-transform property of the heading, thus allowing it to be inherited from its parent element.

Variations

  • The combo class "block" causes the heading to display block, vs. its default of "inline block"
    ‍
    Alternatively, one could have headings default to display block as is more common practice, and add a combo class in cases were they should display inline

  • The combo class "zero" removes top and bottom margins from any heading; this class was necessary for us as our prototyping tool (webflow) implements default margins

Body Text

Body text has the following CSS properties, which are all inherited from the outermost parent (section) level:

  • font weight: 400 (book/normal)
  • font size: 16px
  • line height: 24px

Note that the line-height for body size 16 body copy is 24px, which is different from the line-height of an h6 heading (h6 headings are 16/20)

Hover Interaction

  • Hover: Inner Link Text Wraper
    Bottom Border Color: rgba(33, 33, 33, 0.5)
    Bottom Border Style: Solid (no change)
    Bottom Border Width: 1px (no change)
  • Hover Over: Inner Link
    Nested Icon: Move Right 5px (900ms ease-out-back)
  • Hover Out: Inner Link
    Nested Icon: Move to Origin (600ms ease-out-back)

Element Notes

  • The parent link wrap element is optional.
    It exists to ensure that text link blocks always appear 25x below the max height of the sibling element above. There are a few instances where this spacing is not desired (for example, in the flipper banner). In these instances, the link wrap element is simply removed.
  • The inner link element is necessary.
    It allows both the link text and the arrow icon to be clickable and to direct users to the same place
  • The inner link text wrapper is necessary for hover.
    ‍    As noted in the "hover interaction" section to the left

Variations



the class "arrow" noted above does not inherently change the color of the arrow icon, thus allowing it to inherit its color from the parent element



the class "red-arrow" to the arrow text element instead of just "arrow" as noted above generates this red arrow



the combo class "arrow.inherit-red" will also result in a the red arrow



in cases where the arrow appears before the heading within the link block (i.e. on a procedure page); the combo class "first" is applied, which removes the 10px margin from the left, and adds it to the right: either "red-arrow.first" or "arrow.inherit-red.first"
